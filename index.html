What does version control do?

  • Ensures a coherent view of a complicated system.
  • Records the history of changes to a set of related files (project).
  • Enables navigation of that change history.
  • Supports sharing of the current versions of files and their history among a group of collaborators.

Version control operations

  • checkout/clone: obtain a copy of files
  • update/pull/sync: update local copy to latest version
  • checkin/push/publish: send your copy of files to the central or remote repo
  • commit: record a set of changes
  • branch: create a separate version of files in the repo

Two Kinds of Version Control

  • Centralized: Imposes a "spoke and hub" workflow
    • Examples are CVS and Subversion
  • Distributed: Each contributor has their own complete history of the project.
    • Examples are Git and Mercurial

What is Git?

  • Git is a distributed version control system.
  • Original author is Linus Torvalds, of Linux fame.
  • "I'm egotistical, and I name all my projects after myself."
  • git means "unpleasant person" in British slang.

What is GitHub?

  • GitHub is a Git repository hosting service.
  • Adds services on top of Git like issues, wikis, and social networking.
